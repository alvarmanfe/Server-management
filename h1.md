## **c) Ansible**

Ihan aluksi testataan SSH:n toimivuus ilman Ansiblea.
```
ssh localhost
```
kerta toimii niin kirjaudutaan nyt saman tien ulos
```
exit
```
Sudolla päivitetään paketit:
```
sudo apt update
```
ja asennetaan Ansible ja tarkastetaan sen versio ajamalla ```ansible --version```

![Sieppaa2](https://github.com/user-attachments/assets/d5736e17-8467-4c5c-aa59-3a703d0104b2)

Luodaan ansible projekti. Tässä vaiheessa minulla oli jo valmiina Ansible directory joten ```cd ansible```. Kirjoitetaan lista hosteista jota tullaan ohjaamaan. Luodaan hosts.ini ja lisätään siihen localhosti. Testataan Ansible SSH:n yli, joka näyttää toimivan. Tein yml tiedoston jonka sisään rooliksi annoin "hello". Loin myös hakemistorakenteen ```roles/hello/tasks```. Roolin sisään luotiin main.yml-tiedosto, joka sisältää varsinaisen tehtävän. Tehtävänä oli kopioida teksti tiedostoon /tmp/hello-ansible. Tämä tehtiin Ansible-moduulilla copy, joka määriteltiin seuraavasti:
```
- copy:
    dest: /tmp/hello-ansible
    content: "Hei maailma Ansiblella!\n"
```
Ajettiin playbooki uudestan komennolla 
```
ansible-playbook site.yml
```
Ansible suoritti roolin onnistuneesti ja loi tiedoston kohdekoneelle.

Lopuksi varmistettiin lopputulos SSH-yhteyden yli komennolla:
```
ssh localhost 'cat /tmp/hello-ansible'
```
Tämä tulosti:
```
Hei maailma Ansiblella!
```
Ansible onnistuneesti suoritti infrastruktuurin hallinnan SSH:n yli ja loi halutun lopputilan kohdekoneelle.
